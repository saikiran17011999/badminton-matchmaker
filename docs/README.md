## Overview

The Badminton Event Matchmaking SaaS follows a clean architecture pattern with clear separation between:

- **Presentation Layer** (Frontend)
- **API Layer** (Controllers/Routes)
- **Business Logic** (Services/Matchmaking Engine)
- **Data Layer** (Models/Database)
